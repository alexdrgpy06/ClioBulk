## 2024-05-18 - Direct DOM Mutation for High-Frequency Zustand Updates
**Learning:** In React components that subscribe to high-frequency state updates (like a global progress bar updating rapidly during batch processing), using standard Zustand hooks (`useStore(state => state.progress)`) causes excessive and unnecessary React re-renders, severely bottlenecking the main thread.
**Action:** Bypass the React render cycle entirely for these specific values. Instead, use `useRef` to get a reference to the DOM node, and within a `useEffect`, subscribe directly to the store (`useStore.subscribe`) to manually mutate the DOM node's properties (e.g., `style.width`, `aria-valuenow`). Ensure you also read the initial state (`useStore.getState()`) before subscribing to prevent visual flickering on mount.
