## 2024-05-24 - Range Input Accessibility Improvements
**Learning:** Standard HTML5 `<input type="range">` elements inherently lack sufficient context for screen readers when representing complex values (like percentages or multipliers) and lack visible focus states in default styling. Adding `aria-valuetext` drastically improves the screen reader experience, and ensuring `focus-visible` classes are present is critical for keyboard navigators, especially in dense settings panels.
**Action:** When working with custom or unstyled range inputs, always verify that explicit labels are linked via `htmlFor`/`id`, apply `aria-valuetext` to announce the human-readable format, hide redundant visual text with `aria-hidden="true"`, and enforce `focus-visible` styling for robust accessibility.
