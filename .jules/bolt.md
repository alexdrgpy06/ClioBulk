## 2024-05-19 - Bypass React render cycle for high-frequency progress updates
**Learning:** In standard Zustand setups, subscribing to highly frequent state updates (like a progress counter during bulk processing) causes unnecessary React component re-renders. The `ProgressBar` component was subscribing to `progress`, which meant it was re-rendering and triggering reconciliation hundreds or thousands of times during a batch operation, blocking the main thread and slowing down overall UI responsiveness.
**Action:** For high-frequency UI updates where only simple DOM styling or attributes change (like progress bars or sliders), decouple from React's declarative render cycle. Instead, subscribe only to the boolean flag controlling the component's mount state (e.g., `processing`), and use `useRef` combined with a direct `useStore.subscribe((state) => { ... })` within a `useEffect` to manipulate the DOM element's `style` and `aria-*` attributes directly. This achieves O(1) DOM updates without React overhead.
