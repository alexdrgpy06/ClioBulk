## 2024-05-24 - High-Frequency Tauri Events Cause O(N²) Bottlenecks
**Learning:** The Tauri backend emits numerous progress events for bulk operations. Previously, the React frontend (`src/App.jsx`) performed an O(N) linear array search (`files.find()`) inside the `process-progress` event listener to update file statuses. When dealing with large batches of files, these high-frequency O(N) lookups within the event handler resulted in an overall O(N²) time complexity, creating a severe performance bottleneck and blocking the main thread.
**Action:** When handling high-frequency events (like progress updates or real-time sockets) that need to correlate with a specific item in a list, always use an O(1) lookup mechanism. In React, a `useRef(new Map())` that is pre-populated before the operation begins is an effective, non-rendering way to track these items by their unique key (like a file path).
