## 2024-05-15 - High-Frequency UI Updates via Zustand\n**Learning:** High-frequency UI updates (like progress bars) driven by Zustand state cause performance bottlenecks due to excessive React re-renders.\n**Action:** Bypass React render cycles by subscribing directly to the Zustand store (`useStore.subscribe`) and using a `useRef` to directly manipulate DOM styles (e.g., `width`). Ensure ARIA attributes are updated on the same node.
